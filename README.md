# 大学入試調査マルチエージェントシステム

Claude Code のサブエージェント（Agent）・Skill機能を使い、指定した複数の大学の「入試概要」と「入試制度の変更点（経年比較）」を調査し、統合レポート（Markdown）を自動生成するマルチエージェントシステムです。スライド化は本システムの範囲外とし、生成されたMarkdownレポートを「Claude for PowerPoint」等のClaudeのファイル作成機能に直接アップロードして行う運用とします（詳細は「出力構成」欄の下の節を参照）。

## 想定読者・用途

- 教育業界への転職活動における業界研究資料
- 教育事業者（学習塾・予備校等）の経営判断材料
- 学習塾の現場社員が進路指導時に参照する資料

## アーキテクチャ

| 役割 | 定義ファイル | 担当業務 |
|------|------------|---------|
| PM（メインエージェント） | `CLAUDE.md` | タスク分解・大学リストや調査範囲の確定・進捗管理・最終統合 |
| リサーチャー | `.claude/agents/researcher.md` | 大学ごとの入試調査（対象校数ぶん並列実行） |
| プランナー | `.claude/agents/planner.md` | 調査データをもとにレポート構成案を作成 |
| ライター | `.claude/agents/writer.md` | レポート執筆・`exam-report-format` Skillでの整形 |
| レビュアー | `.claude/agents/reviewer.md` | 成果物の品質チェック・合否判定 |
| （フォールバック）スライド生成 | `.claude/agents/slide-builder.md` | `md-to-slides` Skillでのpptx生成。標準フローでは使用しない |

処理フロー（標準）：

```
[researcher × N（大学ごとに並列）]
        ↓
    planner（構成案作成）
        ↓
     writer（執筆・整形）
        ↓
    reviewer（品質チェック）
        ↓
  （完成したMarkdownレポートをユーザーがClaude for PowerPoint等に
    直接アップロードしてスライド化。本システムはここまで）
```

pptx自動生成が必要な場合のみ、ユーザーが明示的に指示することでslide-builderをフォールバックとして呼び出せます。

## 特徴：調査対象は都度プロンプトで指定可能

対象大学の数・大学名・学部/学科の粒度・調査対象年度範囲は固定値ではなく、ユーザーがプロンプトで指定します。例：

```
早稲田大学・慶應義塾大学・上智大学の3校について、
文系主要学部（法学部・経済学部・商学部/経営学部）を対象に、
過去3年間の入試制度の変更点を含めて調査してください。
```

大学数が変われば、PMが起動するresearcherサブエージェントの数もそれに応じて増減します。

## スライド化は本システムの範囲外

以前はmd-to-slides Skillでpptxを自動生成していましたが、現在は標準フローに含めていません。Claudeのファイル作成機能（「Claude for PowerPoint」等）はWord・PDF・Markdownファイルを直接アップロードできるため、reviewer承認後のMarkdownレポートをそのままアップロードしてスライド化すれば十分であり、pptxを自動生成してから読み込ませる二度手間を避けられるためです。

`slide-builder` エージェントと `md-to-slides` Skillはリポジトリに残していますが、ユーザーが明示的に「pptxも生成して」と指示した場合のみのフォールバック用途です。

## 出力構成

| 種別 | 出力先 |
|------|--------|
| 各大学の調査データ | `tmp/researcher/{大学名}.md` |
| 統合構成案 | `tmp/planner/structure.md` |
| 統合レポート（最終成果物） | `output/{テーマ名}/reports/{テーマ名}.md` |
| （フォールバック時のみ）スライド（pptx） | `output/{テーマ名}/slides/{テーマ名}.pptx` |
| （フォールバック時のみ）スライド画像 | `output/{テーマ名}/images/slide-NN.png` |

レポートは5セクション構成（エグゼクティブサマリー／入試方式の全体像／学部・学科別 入試詳細／入試制度の変更点（経年比較）／受験生・進路指導への示唆）で、各セクション内で対象大学を横断比較する形式になっています。沿革・キャンパス概要等の静的な大学基本情報は独立セクションとして扱わず、学部のキャンパス移転・改組等の変更のあった事実のみ「入試制度の変更点」に含めます。

## セットアップ

- Claude Code（本リポジトリを開いて `.claude/agents` `.claude/skills` を認識させる）
- （フォールバックのpptx生成を使う場合のみ）Node.js v20以上、Python 3 + Playwright

## 実行方法

Claude Code でこのディレクトリを開き、調査したい大学・学部・年度範囲をプロンプトで指定して依頼してください。PMエージェントが `CLAUDE.md` の定義に従い、フェーズ0（対象確定）→ フェーズ1（並列調査）→ フェーズ2（構成・執筆・レビュー）の順に自動で進行し、Markdownレポートの完成をもって完了します。生成されたレポートは、ユーザー自身がClaude for PowerPoint等にアップロードしてスライド化してください。

## 由来

本プロジェクトは、侍エンジニアの学習教材で作成したマルチエージェント練習用リポジトリ（企業の市場調査を行うシステム）をベースに、大学入試調査向けに役割定義・Skillテンプレートを再設計したものです。`md-to-slides` Skillはファイル自体は残していますが、標準フローからは外し、フォールバック専用としています。
